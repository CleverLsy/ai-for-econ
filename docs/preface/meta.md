# 元说明：本书是怎么用 AI 写的

这本手册的写作过程本身就是一个 AI 协作案例。这一节如实记录每一次迭代——**用了什么工具、AI 做了什么、人做了什么、踩了什么坑**。

不只是为了透明，更是为了让读者直观看到：**人机协作不是"AI 写完一键上线"，而是不断的对话、决策、修正**。

## 工具栈

- **Harness**：Hermes Agent + 飞书网关（手机/电脑跨平台输入）
- **模型**：Claude Sonnet
- **静态站**：MkDocs Material
- **代码托管**：GitHub
- **写作环境**：终端（深度工作）+ 飞书（碎片输入）

## 写作工作流

```mermaid
graph LR
    A[人提想法<br/>飞书/终端] --> B[AI 提澄清问题<br/>+ 提建议]
    B --> C[人拍板方向]
    C --> D[AI 起草内容]
    D --> E[人审阅修改<br/>提隐私/风格修正]
    E --> F[AI 重写/补充]
    F --> G[本地构建预览]
    G --> H[Git 提交 + 推送]
    H --> I[更新本元说明]

    style B fill:#1976d2,color:#fff
    style E fill:#d32f2f,color:#fff
    style I fill:#388e3c,color:#fff
```

**核心原则**：

- AI 不是写作工具，是**协作者**
- 每次 AI 输出后，人**必须读**，不能盲推
- 隐私边界由人画，AI 默认按"博客面向公开读者"来处理
- 元说明持续更新——**每一次 AI 协作的迭代都写下来**

## 迭代日志

### v0.1 — 初始化（2026-05-27）

**触发**：作者想写一份"AI for Econ"博客系列，给所有经济学研究者看。

**对话过程**：

1. **brainstorm 阶段**：作者列出三模块大纲（概念/工具/场景），AI 补充：
    - 修正名词拼写（"honey's engineering" → "harness engineering"，"Cloud Code" → "Claude Code"）
    - 补充缺失概念（token、上下文窗口、幻觉、Vibe Coding、eval）
    - 提出读者画像问题（决定写作语气）
    - 提出"国内访问性"维度（很多研究者用不了 Claude）

2. **决策阶段**：作者拍板"面向所有经济学研究者的博客 + 系列文章 + 网站 + 案例"。AI 提出仓库选型：
    - 仓库方式（A 独立 / B 挂主页）→ 选 A
    - 静态站方案（MkDocs / Quarto / VitePress / Docusaurus）→ 先选 MkDocs
    - 大纲是否调整 → 7 大模块定稿

3. **二轮讨论**：作者问"Quarto 在数据可视化上更好吗"，AI 给了 Quarto vs MkDocs 的对比表，作者最终决定：
    - 主体用 MkDocs Material（导航、搜索强）
    - 案例库章节用 mkdocs-jupyter 插件嵌入 .ipynb（拿到可执行代码）

4. **执行阶段**：
    - 建立仓库骨架（49 个占位文档）
    - 配置 mkdocs.yml（中文主题、导航、Mermaid、数学公式）
    - 写 GitHub Actions（自动部署）
    - 同步写 demo 单页（约 1.4 万字）
    - 本地构建验证（HTTP 200，0.46s 无错）

**人机分工**：
- 大纲、读者画像、仓库选型、隐私边界 → **人**
- 大纲展开、占位文档生成、配置文件、demo 内容初稿 → **AI**
- 验证、决策点、风格统一 → **人**

**踩的坑**：
- pip 用官方源装 mkdocs 包卡住 5 分钟无输出，换清华源秒装
- 一开始 mkdocs.yml 用了"deep purple"主题色但中文字体没指定 Noto Sans SC，AI 主动加上

### v0.2 — 概念地图全章完工（2026-05-27）

**目标**：把概念章 8 篇全写完（index + 1.1-1.7），作为博客第一篇可发布的内容。

**AI 主导部分**：
- 7 概念关系图（Mermaid）
- 每章固定结构：一句话理解 → 详细展开 → 经济学场景例子 → 给经济学研究者的核心要点
- 大量真实场景对比（"用 vs 不用" 的具体效果差异）
- 每章末尾的"起步建议"（如必装 3 个 MCP、第一批 5 个 Skill）

**人主导部分**：
- 章节顺序与详略权衡
- 隐私清理（删除真名、学校、个人主页、本地路径、端口号）
- 决定不在这一轮做学术工具的深度对比（留到工具栈章节）

**总产出**：4.6 万字，构建 0.66s 无错误。

**踩的坑**：
- 第一版多处带个人信息（真名、学校/职位、个人主页域名、本地路径）
- 作者一句"删掉里面我的学校信息，以及所有隐私信息，端口信息也不要"触发完整清理
- AI 用 search_files 做了一次全仓扫描，确认零命中后才提交
- **教训**：以后**第一次写带身份的页面就用泛化身份**（如 "CleverLsy，经济学研究者"），不要先写真名再删

### v0.4 — 写在前面章节完工（2026-05-27）

**目标**：把 `preface/index.md` 从占位符改写为正式内容。

**AI 主导部分**：
- 将作者原始设计思路（三模块大纲 + 玩具阶段定位）改写为正式引子
- 底部添加子章节导航卡片，与首页风格一致

**人主导部分**：
- 提供原始文字素材（三模块说明 + 项目定位）
- 决定补充进"写在前面"而非首页

**人机分工**：原始措辞 → **人**；结构化改写、卡片布局 → **AI**

### v0.3 — 首页排版优化（2026-05-27）

**目标**：改善 `extra.css` 排版细节，修复 `index.md` 合并冲突。

**AI 主导部分**：
- 分析 CSS 中 6 处排版问题并一次性修复
- 解决 `index.md` 的 Git 合并冲突

**人主导部分**：
- 提出"帮我全改了"的授权
- 确认推送

**踩的坑**：
- `index.md` 存在未解决的 `<<<<<<` 合并冲突标记，导致首页渲染异常，需手动清理
